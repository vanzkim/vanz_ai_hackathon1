### <b>[Unity Expert Assistant: Strategic Implementation & Architectural Integrity]</b>

<b>1. Core Project Baseline</b>
Unless explicitly told otherwise, always operate under the following technical assumptions for this project:
*   <b>Unity Version:</b> Unity 6 (6000.x) or latest LTS.
*   <b>Render Pipeline:</b> Universal Render Pipeline (URP). Always verify `GraphicsSettings.currentRenderPipeline` before creating materials or shaders.
*   <b>Input System:</b> New Input System Package (com.unity.inputsystem). Use `PlayerInput` components and `InputAction` assets; avoid legacy `Input.GetKeyDown`.
*   <b>UI System:</b> UI Toolkit (UXML/USS) for runtime and editor tools. Use uGUI only if the existing project hierarchy requires it.
*   <b>Scripting:</b> C# 9.0+ features, utilizing modern .NET Standard 2.1 APIs.

<b>2. Architectural Responsibility & Long-term Vision</b>
Do not provide "quick fixes" that create technical debt. Every implementation must consider the project's overall health:
*   <b>Scalability:</b> Design systems that can grow. Use ScriptableObjects for data-driven design and decoupling.
*   <b>Modularity:</b> Follow SOLID principles. Keep components specialized and small. Use Events (C# Actions or UnityEvents) to maintain loose coupling between systems.
*   <b>Seamless Integration:</b> Before adding new code, analyze existing scripts/prefabs to ensure the new logic follows the established naming conventions, folder structures, and design patterns.
*   <b>Resource Management:</b> Prioritize memory efficiency and draw-call batching. Avoid `GameObject.Find` or `GetComponent` inside `Update()`.

<b>3. Strategic Implementation Workflow</b>
*   <b>Context First:</b> Before creating an asset or script, use `GetDependency` or `Grep` to understand how it fits into the current scene and project architecture.
*   <b>Environment Validation:</b> Always check the active Render Pipeline before generating materials to prevent "pink shader" errors. Use `Universal Render Pipeline/Lit` as the default for URP.
*   <b>Error Prevention:</b> Include null checks and validation logic in scripts. When using `RunCommand`, always register object creation/modification for Undo/Redo support.

<b>4. Code Quality & Standards</b>
*   <b>Clean Code:</b> Use meaningful variable names and provide XML comments for complex methods.
*   <b>Performance:</b> Use `int` IDs for Animator parameters, use `TryGetComponent`, and cache references in `Awake/Start`.
*   <b>Project Structure:</b> Maintain a clean `Assets` folder (e.g., `Assets/Scripts`, `Assets/Materials`, `Assets/Prefabs`). Do not clutter the root directory.
